nano.rust

A pure-Rust, semantics-first NanoAOD analysis framework for high-energy physics — built for the agentic-coding era.

The thesis: soft guardrails (prompts, skills, harnesses, review) can steer agent-generated analysis code but cannot guarantee its correctness. A hard guarantee needs a mechanical enforcer — so we model the analysis as a typed state machine and let the Rust compiler make invalid analysis states unrepresentable, then add Rust's strengths: performance, FFI to legacy libraries, SIMD per-event execution, and a TUI for orchestration and agentic execution.

What works today

  • Pure-Rust ROOT I/O — reads & writes TTree (zlib/LZMA/lz4/zstd), no ROOT/C++ dependency.
  • Real CMS NanoAODv9 — reads it locally and remotely on-demand over HTTPS byte-range (10 events ≈ 1.3 MB of a 2 GB file).
  • Bounded streaming — real NanoAOD → skim at ~72 MB resident, any file size.
  • uproot-validated — the reader is value-correct against uproot on real data; CI cross-checks every push.
  • Typed event model — collections, attributes, and a compile-time analysis state machine (in design).
